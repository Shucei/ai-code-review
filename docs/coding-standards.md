# 代码规范文档

本文档基于前端项目的实际代码规范整理，用于 AI Code Review 检查。

**规则**: 常量必须全部大写，单词之间使用下划线分隔

**规则**:

- 普通函数使用小驼峰命名法 (camelCase)
- React 组件使用大驼峰命名法 (PascalCase)

**规则**: 枚举必须显式初始化

**规则**: 避免使用魔法数字，应定义为常量

**允许的数字**: -1, 0, 1

**规则**: 优先使用可选链操作符

**规则**: 无子元素的组件必须使用自闭合标签

**规则**: Props 中不应使用不必要的大括号

**规则**: 列表渲染必须提供唯一的 key

**规则**: 避免在 Context.Provider 中使用构造的对象值

**规则**: 函数参数不应超过 3 个，尽量用对象形式

**规则**: 对象属性和方法应使用简写语法

**规则**: 优先使用解构赋值

**规则**: 优先使用扩展运算符而不是 `apply`

**规则**: 优先使用对象展开而不是

**规则**: 某些大型库必须通过动态导入使用
